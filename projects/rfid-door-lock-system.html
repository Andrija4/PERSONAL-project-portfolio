RFID Door Lock System For Faculty's Laboratory alongside logging web service

IoT 2026

The facility needed better solution for accessibility to the laboratory. Instead of using key lock for laboratory door (which was outdated based on the purpose), professor assigned me a task of making RFID door lock alongside webservice that would track who has accessed the laboratory.

HTML CSS HTMX SQLAlchemy SQLite Python 3.12 ESP32 RC522 FastAPI Electronics
View on GitHub

System Architecture


Hardware Layer:
Used ESP32 microcontroller alongside RC522 sensor, led diodes, buzzer, power supply(12V/1A), relay and YE-304(NO) solenoid door lock. Based on a fact that the whole system had to be assembled from zero, I was also in charge of doing all the wirings and soldering.


Communication:
ESP32 sends GET request for getting valid RFID cards and POST for tracking when was the card used.


API:
For communication between database and service I used FastAPI which I was already familiar with


Power Management:
ESP32 together with solenoid door lock are powered through power plugs near the door. In case power goes off, solenoid door gets opened and stays that way till power comes back.


Frontend:
Frontend was built using HTMX and Jinja2 templates.


Backend:
Backend: FastAPI handles API requests and responses. For database operations, I utilized SQLAlchemy and SQLite.


Challenges & Solutions


• Challenge:
- Making sure that if power goes out, someone can still access laboratory.

• Solution:
- Used solenoid (NO) - Normally Open door lock.
